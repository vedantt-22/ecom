---Auth.Middleware---

1. requireAuth Function
This function checks if the user is Authenticated (Who are you? Do you have a valid ID card/JWT?).

Line-by-Line Breakdown:
passport.authenticate("jwt", { session: false }, ...): We tell Passport to use the JWT strategy. We set session: false because we aren't using old-school cookies/sessions; we are using modern Tokens.

The Callback (err, user, info):

err: If the database crashes or something explodes, we return a 500 Internal Server Error.

!user: If the token is expired, fake, or missing, we return a 401 Unauthorized. We send a JSON message so your Angular frontend knows exactly why the login failed.

req.user = user: This is a crucial step. If the user is valid, we "attach" them to the request object. Now, every following function knows exactly which user is making the request.

next(): This tells Express: "Everything looks good, move to the next function in line."

)(req, res, next): This looks weird, right? passport.authenticate actually creates a function. This final part immediately executes that function with the current request details.

2. requireRole Function
This is a Middleware Factory. It doesn't just check if you are logged in; it checks if you are Authorized (Are you an Admin? A Customer?).

export function requireRole(...allowedRoles: string[]): The ... is a "rest parameter." It allows you to pass one or many roles (e.g., requireRole("admin", "manager")).
return (req, res, next) => { ... }: This function creates another function. We do this so we can "save" the allowedRoles inside the middleware.
const user = req.user as any: It grabs the user we attached earlier in requireAuth.
!allowedRoles.includes(user.role): It checks: "Is the user's role in the list of roles allowed for this specific route?"
res.status(403): If a regular customer tries to enter the /admin panel, they get a 403 Forbidden.
Note: 401 means "I don't know who you are." 403 means "I know who you are, but you aren't allowed in here."
